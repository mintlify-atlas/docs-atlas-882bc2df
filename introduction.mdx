The `@lando/drupal` plugin is the official [Lando](https://lando.dev) integration for [Drupal](https://www.drupal.org/). It provides dedicated recipes for every supported Drupal version — from the legacy Drupal 6 through the current Drupal 11 — so you can spin up a fully containerized local environment without installing PHP, Composer, or a database server globally on your machine.

## Supported Drupal versions

The plugin ships recipes for the following Drupal versions:

| Recipe | Drupal version | Status |
|---|---|---|
| `drupal11` | Drupal 11 | Supported |
| `drupal10` | Drupal 10 | Supported |
| `drupal9` | Drupal 9 | Supported |
| `drupal8` | Drupal 8 | Legacy (EOL) |
| `drupal7` | Drupal 7 | Legacy (EOL) |
| `drupal6` | Drupal 6 | Legacy (EOL) |

<Note>
Drupal 8, 7, and 6 are end-of-life and no longer receive upstream security updates. Their Lando recipes are maintained for legacy use only. See the [Legacy Versions](/legacy/drupal-8) section for details.
</Note>

## How it works

The plugin builds on top of Lando, which uses Docker and Docker Compose to orchestrate containers. When you run `lando start`, Lando reads your `.lando.yml`, picks the appropriate `drupal*` recipe, and starts the configured set of services — a PHP appserver, a database, and optionally a separate nginx container.

A minimal Drupal 11 Landofile looks like this:

```yaml .lando.yml
name: my-cms
recipe: drupal11
config:
  php: '8.3'
  via: apache:2.4
  database: mysql:8.0
```

All services are destroyed and recreated cleanly with `lando destroy` and `lando start`, making it straightforward to reset your local environment to a known state.
